Openstack has released its latest version Icehouse. The ninth release from the popular open source cloud program, has 2,902 bug fixes and over 350 new features including a new database service, support for Hadoop deployments on OpenStack clouds, and improved support for container. Here's an overview of Icehouse features.

OpenStack Compute (Nova):

Additional support for rolling upgrades means minimal impact to the existing/running workloads during the upgrade process. The new version also requires third party drivers to adopt much more stringent testing requirements. Other improvements include: improved access to the data layer for efficient performance, especially at scale; improved boot process reliability; API updates to apprise end users about new features.

OpenStack Object Storage (Swift):

This significantly improves workflows and saves time by allowing users to find out what capabilities are available via API call. Users can simply ask Object Storage cloud what capabilities are available via API call. The introduction of Ssync allows more efficiency in transporting data, significantly improving performance.

OpenStack Block Storage (Cinder):

There is more consistency in user experience across storage platforms with mandatory testing for external drivers. Fully distributed services improve scalability. There are several updated added for backend migration with tiered storage environments, allowing for performance management in heterogeneous environments.

OpenStack Networking (Neutron):

Performance of provisioning actions is improved with tighter integration with OpenStack Compute. It also ensures consistency with bulk instance creation. There is consistency and reliability across network implementation with better functional testing for actions that require coordination between multiple services, and third-party driver testing.

OpenStack Identity Service (Keystone):

First iteration of federated authentication is now supported allowing users to access private and public OpenStack clouds with the same credentials.

OpenStack Orchestration (Heat):

There is more lifecycle management for applications with the new configuration of API. It also brings in new capabilities to end users that were previously limited only to cloud administrators. Also automated scaling of additional resources across the platform, including compute, storage, and networking is now available. Collaboration with OASIS resulted in the TOSCA Simple Profile in YAML v1.0, demonstrating how the feedback and expertise of hands-on OpenStack developers can dramatically improve the applicability of standards.

OpenStack Telemetry (Ceilometer):

Improved access to metering data used for automated actions or billing/chargeback purposes.

OpenStack Dashboard (Horizon):

The Dashboard is now available in 16 languages, including German, Serbian, and Hindi. Design is updated with new navigation and user experience improvements (e.g., in-line editing).
